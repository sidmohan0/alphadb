# X Fail-Closed Budget Gates

Strict fail-closed X API budget-gate machinery is out of scope for the current lean research workflow.

## Why this is out of scope

The current research mode needs direct, low-friction data pulls over explicit time ranges. The real X run cost only a few dollars and was controlled by an explicit daily cap and human intent. Additional approval ledgers, spend-to-date fields, and multi-step budget gates add process overhead without improving the immediate research decision.

This does not reject basic cost visibility. Estimates, actual request counts, and actual cost summaries remain useful. The rejected scope is bureaucratic gating that blocks straightforward collection when the maintainer has already approved the run.
