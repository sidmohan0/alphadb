Polymarket Discovery Runs — Target Backend Architecture

Visual plan for the next pivot: async run model in Postgres + Redis, while keeping controller logic thin and service-layer orchestration explicit.

DB: Postgres Cache: Redis Primary API: POST/GET /runs Legacy compatibility: GET /market-channels

Layered Backend Map

HTTP Controllers

Parse/validate request params, call services, map structured responses.

polymarket.controller.ts GET /runs/:id, POST /runs, GET /runs/latest, legacy GET /market-channels

Service Layer

Orchestrates create/attach, dedupe keys, concurrency slotting, job lifecycle, cache/read-through, polling behavior.

discovery-run.service.ts

Infrastructure

Postgres for durable state, Redis for distributed locks/semaphores, short-lived responses for UI fast-path.

discoveryRun.repository.ts run-cache.ts

Domain Discovery

Existing extraction logic: fetch `/markets`, parse tokens/assets, optional websocket probing, timeout/error mapping unchanged.

Job Execution

Runner function invoked when run is created or dequeued. Produces channel rows + ws scan summary and finalizes run state.

jobs/executeDiscoveryRun.ts

Request Flow (Primary Endpoints)

1) Create

  • Client: POST /api/polymarket/market-channels/runs
  • Controller normalizes query/body and passes DTO
  • Service computes dedupeKey

2) Concurrency + Dedupe

  • Check Redis active lock by dedupeKey
  • Acquire semaphore (default limit 4)
  • Attach to existing run if in-flight duplicate

3) Persist Run

  • Create discovery_runs row with status queued
  • Write cache pointers: latest run + active dedupe key
  • Return runId + pollUrl as 202

4) Execute + Read

  • Worker marks running and executes discovery
  • Persist channels + ws scan + error metadata
  • GET endpoint reads run + paginated channels

Run State Lifecycle

queued

Created, waiting for worker slot or existing run resolution.

running

Active fetch + websocket probe in progress; channels may be partially available.

succeeded

All tasks complete. Channels+scan fully materialized.

failed

Error recorded in structured format with code/message/retryable.

Suggested Endpoints

POST /api/polymarket/market-channels/runs
Create or attach run. Returns 202 + runId.
GET /api/polymarket/market-channels/runs/:runId?offset=0&limit=200
Run status + paginated channels + ws scan.
GET /api/polymarket/market-channels/runs/latest
Latest run in default scope for dashboard bootstrap.
GET /api/polymarket/market-channels
Compatibility wrapper; keep thin and delegate to run service with optional short wait.

Legacy/Compatibility Strategy

  • Controller remains thin; no orchestration in route handlers.
  • Old GET uses same service as new POST endpoint.
  • Optional waitMs lets GET return immediate terminal payload when run already ready.
  • If not complete, return the same async shell used by runs.
  • Structured error contract stays unchanged: code, message, retryable, details, requestId.

Postgres & Redis Shapes (High-level)

Tables

TablePurposeKey Columns
discovery_runs Run lifecycle, status, source metadata, timings, error summary, retention id, dedupe_key, status, clob_api_url, chain_id, ws params, requested_at, started_at, completed_at, market_count, market_channel_count, error_code, request_id
discovery_run_channels Normalized channel output for pagination discovery_run_id, asset_id, condition_id, question, outcome, market_slug
discovery_run_ws_scans Single ws summary row per run discovery_run_id, ws_url, connected, observed_channels[], message_count, sample_event_count, errors
discovery_run_events (optional) Debug/progress/audit trail discovery_run_id, event_type, event_at, message, metadata

Redis Keys

KeyUseTTL
pm:discovery:latest:default Latest known run id 10m
pm:discovery:run:{runId} Cached run payload/read model 5–30m
pm:discovery:active:{dedupeKey} Distributed dedupe lock (in-flight) max runtime
pm:discovery:active-count Semaphore for concurrent run limit same as run duration or ephemeral

Indexing Focus

  • discovery_runs(id) PK
  • discovery_runs(status, requested_at DESC) for latest/recent views
  • discovery_runs(dedupe_key, status) for active dedupe checks
  • discovery_runs(expires_at) for cleanup jobs
  • discovery_run_channels(discovery_run_id, asset_id) unique per run
  • discovery_run_channels(discovery_run_id) for paginated listing
