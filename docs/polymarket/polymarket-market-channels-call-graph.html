3) Is run() at the bottom intentional?

Not coincidental. In JS/TS, function declarations are hoisted, so run() can be called even if it is defined near the bottom. It is a style/readability choice: orchestration is often kept near the end, while helpers live above or below.

For a runtime trace during a failure, you want a stack trace (runtime call stack). What is shown above is a static call graph.
