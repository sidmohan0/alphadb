## Conventions

Use the Linear connector/app when available. If a skill says "publish to the issue tracker", create a Linear issue in team `alphadb`.

When creating issues:

- Put each independently actionable work item in its own Linear issue.
- Include acceptance criteria.
- Include links to relevant repo files, research docs, or ADRs when available.
- Apply triage labels/statuses using `docs/agents/triage-labels.md`.

When reading issues:

- Prefer the Linear issue body, comments, labels, status, and linked project context.
- If a Linear issue references repo docs, read those docs before implementation.

If Linear tooling is unavailable, ask the user whether to continue with local markdown notes or wait for Linear access.
