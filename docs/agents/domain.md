# Domain Docs

This is a single-context repo.

Before product or architecture work:

- Read `CONTEXT.md`.
- Read `docs/architecture/boundaries.md` for Cockpit, AlphaDB API, Operational State, Runtime, Replay/Research, Data Explorer, Lab, and agent boundaries.
- Read root `docs/adr/` if ADRs exist.
- Keep new domain terms in `CONTEXT.md`.
- Add ADRs only for decisions that are hard to reverse, surprising without context, and the result of a real trade-off.
